May 1, 2020

2020 年 5 月 1 日

## Dashboard

## 控制台

- 项目设置（Project Settings）中插入的环境变量（Environment Variables）现在支持换行符。您可以直接复制粘贴值，或使用 `Shift` + `Enter` 提交包含换行符的值。

- 我们已上线全新的[联系页面](https://vercel.com/contact)，您可借此便捷地创建技术支持工单，或联系我们的销售团队。

- 悬停在部署 URL 上时，不再显示“Visit”（访问）提示工具栏（tooltip）。取而代之的是，您可在部署详情页直接点击“Visit”（访问）按钮。

- 部署详情页中的“Functions”（函数）标签页现已明确说明：日志为实时流式输出，如需回溯查看（即持久化存储），需将其转发至第三方服务。

- 打开团队设置（Team Settings）时，焦点将不再自动聚焦于团队 URL 输入框。

- 活动动态（Activity feed）中显示的环境变量值现已被截断，以控制显示长度。

- 在命令行接口（CLI）的[下载安装页面](https://vercel.com/download)上，我们现已推荐安装 `vercel` 包，而非已弃用的 `now` 包。

- Hitting the limit of 10 team members on the Pro plan now presents a button for easily contacting the Sales team.

- Pro 计划团队成员数达到 10 人上限时，现在会显示一个按钮，方便您直接联系销售团队。

- Introduced a limit of 100 Environment Variables per Project.

- 每个项目引入了最多 100 个环境变量（Environment Variables）的限制。

- Made "Ionic Angular" or "Docusaurus 2" templates work again.

- 修复了“Ionic Angular”和“Docusaurus 2”模板，使其恢复正常可用。

- Signing up with a Git account that is already connected to a personal Vercel account will now offer you to log in instead of erroring.

- 使用已关联至个人 Vercel 账户的 Git 账户注册时，系统将提示您直接登录，而非报错。

- Signing up with a Git account that has the same email address as a personal Vercel account will now offer you to merge them instead of creating a new one.

- 使用与现有个人 Vercel 账户邮箱地址相同的 Git 账户注册时，系统将提示您合并账户，而非创建新账户。

  - Trying to log in with a Git account that is not connected to a personal Vercel account will now log you into a personal Vercel account that has the same email address.

  - 尝试使用未关联任何个人 Vercel 账户的 Git 账户登录时，若系统中存在邮箱地址相同的个人 Vercel 账户，则将自动登录该账户。

## Domains

## 域名（Domains）

- Removing a domain now triggers a cascade of events that will clean up all of its associated Aliases, Certificates and DNS Records from the system.

- 删除域名现将触发一系列级联操作，自动从系统中清理该域名关联的所有别名（Aliases）、证书（Certificates）及 DNS 记录（DNS Records）。

## Deployments

## 部署（Deployments）

- Selecting "Other" in the list of framework presets, overriding the Build Command and then creating a deployment containing `vercel.json` and a few static files does not cause an error anymore.

- 在框架预设列表中选择“Other”，覆盖构建命令（Build Command），再创建一个包含 `vercel.json` 文件及若干静态文件的部署，将不再引发错误。

## Documentation

## 文档

- 在新的 `redirects` 路由属性的[配置参考](https://vercel.com/docs/configuration#project/redirects)中，新增了一个 `permanent` 属性。

- `routes` 属性的[配置参考](https://vercel.com/docs/configuration#project/routes)现已明确说明：该属性不可与新的路由属性同时使用。

- 为每个系统环境变量均[添加了示例](https://vercel.com/docs/v2/build-step#system-environment-variables)。

## Git Integration

## Git 集成

- `vercel.json` 和 `.vercelignore` 配置文件现已支持 GitHub、GitLab 和 Bitbucket。

## Command Line

## 命令行

- 一个名为 `vercel` 的新包已发布至 [npm](http://npmjs.org/)，该包支持 `vercel.json`、`.vercelignore`，并附带两个可执行二进制文件：`vercel` 和 `vc`。
