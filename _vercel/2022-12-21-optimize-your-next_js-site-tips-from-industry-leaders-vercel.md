Optimizing your Next.js site for performance and efficiency can be complicated, but a good developer toolkit can help. [Hear from some of the experts](https://www.youtube.com/playlist?list=PLBnKlKpPeagll1CCK08EvjqgCq0C_dXZq) from this year’s Next.js Conf to see how you can best use React Server Components, the latest in web UI, powerful layouts, and more to create a world-class website.

优化 Next.js 网站的性能与效率可能十分复杂，但一套优秀的开发者工具包能助你一臂之力。[聆听今年 Next.js Conf 上多位专家的分享](https://www.youtube.com/playlist?list=PLBnKlKpPeagll1CCK08EvjqgCq0C_dXZq)，了解如何最佳地运用 React Server Components、前沿的 Web UI 技术、强大的布局能力等，打造世界级网站。

Next.js is known as a frontend framework, but it has powerful backend capabilities, too. The CEO of Ping, Theo Browne, shares exactly how Next.js can be used as a backend framework in his presentation. [Tune in](https://youtu.be/W4UhNo3HAMw?utm_source=youtube&utm_medium=web&utm_campaign=yt_nextjs_tips) to see how he compares backend tools with other technologies and explores the tight integration between the frontends and backends he builds.

Next.js 常被视为前端框架，但它同样具备强大的后端能力。Ping 公司首席执行官 Theo Browne 在其演讲中详细阐述了 Next.js 如何作为后端框架使用。[点击观看](https://youtu.be/W4UhNo3HAMw?utm_source=youtube&utm_medium=web&utm_campaign=yt_nextjs_tips)，了解他如何将后端工具与其他技术进行对比，并深入探讨其所构建的前后端之间高度紧密的集成关系。

[Build a design system in Next.js with Tailwind](https://youtu.be/T-Zv73yZ_QI?utm_source=youtube&utm_medium=web&utm_campaign=yt_nextjs_tips)

Want to see how other developers would write a frontend component system? Tru Narla, a Software Engineer at Discord, shares that and more. Tru explains how to build a design system using Tailwind, HeadlessUI, and CVA. You’ll even learn how to conditionally render certain styles within the system by having JavaScript control the underlying CSS. [Watch until the end](https://youtu.be/T-Zv73yZ_QI?utm_source=youtube&utm_medium=web&utm_campaign=yt_nextjs_tips) to see how Tru uses Storybook to handle components in isolation.

想看看其他开发者是如何编写前端组件系统的吗？Discord 软件工程师 Tru Narla 将为你揭晓答案。Tru 详细讲解了如何结合 Tailwind、Headless UI 和 CVA 构建设计系统；你还将学到如何通过 JavaScript 控制底层 CSS，实现在系统内按条件渲染特定样式。[请看到最后](https://youtu.be/T-Zv73yZ_QI?utm_source=youtube&utm_medium=web&utm_campaign=yt_nextjs_tips)，了解 Tru 如何借助 Storybook 对组件进行独立开发与测试。

[Explore Nested layouts by example](https://youtu.be/6mQ3M1CUGnk?utm_source=youtube&utm_medium=web&utm_campaign=yt_nextjs_tips)

[通过实例探索嵌套布局](https://youtu.be/6mQ3M1CUGnk?utm_source=youtube&utm_medium=web&utm_campaign=yt_nextjs_tips)

Since it was first released, Next.js has led the way in building JavaScript apps with fast initial page loads.  

自首次发布以来，Next.js 一直引领着具备快速首屏加载能力的 JavaScript 应用开发潮流。

But in the process of speeding up sites by serving decoupled pages, we lost some of the benefits of React's component tree when building nested UIs.  

然而，在通过提供解耦页面来加速网站的过程中，我们在构建嵌套式 UI 时，部分丧失了 React 组件树所带来的优势。

[In this talk](https://youtu.be/6mQ3M1CUGnk?utm_source=youtube&utm_medium=web&utm_campaign=yt_nextjs_tips), the founder of Build UI, Sam Selikoff, shows by example how nested layouts with Next.js 13 bring the benefits of React’s component tree back to the forefront while improving both the DX and UX of your Next.js applications.  

[在本次分享中](https://youtu.be/6mQ3M1CUGnk?utm_source=youtube&utm_medium=web&utm_campaign=yt_nextjs_tips)，Build UI 创始人 Sam Selikoff 将通过实际示例演示：Next.js 13 的嵌套布局如何重新凸显 React 组件树的优势，同时显著提升您 Next.js 应用的开发者体验（DX）与用户体验（UX）。
