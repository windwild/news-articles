Apr 1, 2020

2020 年 4 月 1 日

## Dashboard

## 控制台

- 暗色模式现在将根据您的系统设置自动启用。如需覆盖该行为，请通过头像菜单访问主题切换器进行手动设置。

- 在导入第三方 Git 仓库或模板时，如果系统无法在您已连接的 Git 账户中为您创建 Git 仓库，界面 now 会显示一条有用的提示信息。

- 在导入 Git 仓库时，若缺少必需的配置文件，界面 now 会正确显示相应的错误信息。

- 向项目中添加环境变量后，焦点将自动回到“名称”输入框，以便您更便捷地继续添加更多环境变量。

## Documentation

## 文档

- 已在 [API 参考文档](https://zeit.co/docs/api#endpoints/deployments/cancel-a-deployment) 中新增用于取消构建部署的端点。

- 现在可通过分页方式检索 [项目](https://zeit.co/docs/api#endpoints/projects/get-projects) 和 [密钥](https://zeit.co/docs/api#endpoints/secrets/list-secrets)，相关文档现已同步更新以准确反映该功能。

- 关于 [重写规则（rewrites）](https://zeit.co/docs/configuration#project/rewrites)、[重定向（redirects）](https://zeit.co/docs/configuration#project/redirects) 和 [响应头（headers）](https://zeit.co/docs/configuration#project/headers) 的配置项文档，现已补充说明各项允许的最大条目数量。

## Git Integration

## Git 集成

- 重命名您的个人 GitHub 账户或 GitHub 组织，将不再影响 Git 集成的功能表现。

## Pricing

## 定价

- 旧版 Advanced（高级）套餐已全面下线。

- 所有团队现在均可在「账单」页面自定义发票内容，无需额外开通任何功能。

- 预览部署后缀（Preview Deployment Suffix）现已作为一项附加功能，面向所有订阅 [新版 Pro 套餐](https://zeit.co/pricing) 的团队开放。

## Projects

## 项目

- 在 2020 年 4 月 30 日 00:00（UTC）之后创建的项目中，部署将不再自动获取 Git 环境变量。此后创建的项目如需使用与 Git 相关的信息，应显式配置 [系统环境变量](https://vercel.com/docs/v2/build-step#system-environment-variables)。
