Mar 23, 2020

2020年3月23日

在 Vercel 上创建新项目，只需从您最常用的代码托管平台导入一个 Git 仓库即可——无论该平台是 [GitHub](https://zeit.co/docs/v2/git-integrations/zeit-now-for-github)、[GitLab](https://zeit.co/docs/v2/git-integrations/zeit-now-for-gitlab)，还是 [Bitbucket](https://zeit.co/docs/v2/git-integrations/zeit-now-for-bitbucket)。

项目导入完成后，Git 集成连接可在转瞬之间完成编辑。今天，我们进一步优化了这一流程，使其更易于理解，也比以往更加可靠。

## 编辑项目的 Git 连接

## 编辑项目的 Git 连接

如果您希望管理某个项目的 Git 连接，现在可通过项目设置中的全新 **“Git 集成”（Git Integration）选项** 来实现：

![项目设置中全新的“Git 集成”字段。](images/new-git-integration-settings-vercel/img_001.jpg)  
项目设置中全新的“Git 集成”字段。

有关该新字段的更多详情，请参阅我们的 [文档](https://zeit.co/docs/v2/platform/projects#git-integration)。

## 查看账户的 Git 连接状态

## 查看账户的 Git 连接状态

为帮助您全面了解当前 Vercel 账户已连接至哪个 Git 托管平台，我们对个人账户及团队账户的 **“设置”页面中的“Git 集成”（Git Integrations）页面** 进行了优化。

该页面现已清晰展示 Git 连接的状态，并明确指引您如何编辑该连接：

![The refined "Git Integrations" settings page for personal accounts and teams.](images/new-git-integration-settings-vercel/img_002.jpg)  
针对个人账户和团队优化后的“Git 集成”设置页面。

## Conclusion

## 结论

将您的项目部署到 Vercel，最佳方式是为 [GitHub](https://zeit.co/docs/v2/git-integrations/zeit-now-for-github)、[GitLab](https://zeit.co/docs/v2/git-integrations/zeit-now-for-gitlab) 和 [Bitbucket](https://zeit.co/docs/v2/git-integrations/zeit-now-for-bitbucket) 之一配置我们的 Git 集成。配置完成后，只需执行 `git push` 即可自动创建部署。

自今日起，管理现有 Git 服务商连接的操作，已与导入 Git 仓库一样简单便捷。
