Aug 5, 2024

2024 年 8 月 5 日

![](images/improved-live-mode-in-runtime-logs-vercel/img_001.jpg)![](images/improved-live-mode-in-runtime-logs-vercel/img_002.jpg)

You can now toggle live streaming for [Runtime Logs](https://vercel.com/docs/observability/runtime-logs) to update every ~5 seconds without clearing existing logs or manual refreshes.

现在，您可切换 [运行时日志（Runtime Logs）](https://vercel.com/docs/observability/runtime-logs) 的实时流模式，日志将每约 5 秒自动更新一次，无需清空已有日志，也无需手动刷新。

Runtime logs capture crucial information from server-side rendering, API routes, Vercel Functions, and more. For advanced use cases, you can export logs to external endpoints or [integrations](https://vercel.com/integrations#observability) using [Log Drains](https://vercel.com/docs/observability/log-drains).

运行时日志可捕获来自服务端渲染（SSR）、API 路由、Vercel Functions 等的关键信息。对于高级使用场景，您可通过 [日志导出（Log Drains）](https://vercel.com/docs/observability/log-drains) 将日志导出至外部端点或 [可观测性集成](https://vercel.com/integrations#observability)。

Learn more about [Runtime Logs](https://vercel.com/docs/observability/runtime-logs).

了解更多关于 [运行时日志（Runtime Logs）](https://vercel.com/docs/observability/runtime-logs) 的信息。
