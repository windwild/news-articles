Jul 1, 2020

2020 年 7 月 1 日

## Deployments

## 部署

- 如果存在适当的 `Cache-Control` 响应头，则 Serverless Function 返回的状态码为 `404`、`301` 或 `308` 的响应现在也将被缓存。此前，仅 `200` 状态码的响应会被缓存。

- 每当使用 Next.js 的 [增量静态再生（Incremental Static Regeneration）](https://nextjs.org/blog/next-9-4#incremental-static-regeneration-beta) 功能时，更新内容现在仅在所有 [边缘网络（Edge Network）](http://vercel.com/edge-network) 区域均完成接收后才对外生效。

- 现在可通过在创建新部署（Deployment）时显式包含点文件（dotfiles），使其公开可访问。此前，点文件始终被忽略。

- 将 Serverless Function 部署至多个区域的功能现已 [受限](https://vercel.com/docs/v2/platform/frequently-asked-questions#deploying-serverless-functions-to-multiple-regions)，仅对企业版（Enterprise）计划开放（该限制自新的定价模型 [发布](https://vercel.com/blog/simpler-pricing) 起即已在 [定价页面](https://vercel.com/pricing) 中列出，但直至目前才正式执行）。

- 现在可通过 `deploy.new/<template>` 和 `deploy.new/<owner>/<repo>` 快速导入模板（Template）或 GitHub 仓库。相关文档即将上线。

- 当 [Docusaurus](https://vercel.com/import/templates) 部署的输出目录（Output Directory）缺失时，系统现在会显示一条清晰有用的错误信息。

- 向部署（Deployment）发送反向范围请求头（如 `Range: bytes=5-1`）时，系统现在将返回 `416` 状态码响应，而非此前导致服务崩溃的 `500` 错误。

- 当部署构建过程中触及文件数量上限（16,000 个文件）时，系统现在将显示一条清晰有用的错误提示信息。

- If one of the Serverless Functions created from a Deployment is larger than 250MB unzipped, a useful error message will now be rendered.

- 如果某个由部署（Deployment）生成的 Serverless Function 解压后体积超过 250 MB，系统现在将显示一条清晰有用的错误信息。

- Log lines piped to [Log Drains](https://vercel.com/integrations?category=logging) will now correctly render `vercel` instead of `now` as the origin application name.

- 发送到 [Log Drains（日志接收器）](https://vercel.com/integrations?category=logging) 的日志行，其来源应用名称现在将正确显示为 `vercel`，而非旧有的 `now`。

- The `.nuxt` directory is now cached automatically for Nuxt.js Projects. Additionally, default caching headers are provided for the client assets.

- 对于 Nuxt.js 项目，`.nuxt` 目录现在将自动被缓存。此外，客户端资源（client assets）也将默认附带缓存响应头（caching headers）。

- Placing invalid sub properties within the [redirects](https://vercel.com/docs/configuration#project/redirects), [rewrites](https://vercel.com/docs/configuration#project/rewrites) or [headers](https://vercel.com/docs/configuration#project/headers) properties will now render a useful error message.

- 若在 [重定向（redirects）](https://vercel.com/docs/configuration#project/redirects)、[重写（rewrites）](https://vercel.com/docs/configuration#project/rewrites) 或 [响应头（headers）](https://vercel.com/docs/configuration#project/headers) 配置项中设置了无效的子属性，系统现在将显示一条清晰有用的错误信息。

- Defining a `functions` property with invalid paths will now render a useful error message.

- 若在 `functions` 配置项中指定了无效路径，系统现在将显示一条清晰有用的错误信息。

## Dashboard

## 控制台（Dashboard）

- The [Logs and Source Protection section](https://vercel.com/docs/v2/platform/projects?query=logs%20and%20source%20prote#logs-and-source-protection) in the Project Settings now doesn't require a new Deployment in order for the setting to be applied anymore.

- 项目设置（Project Settings）中的 [日志与源码保护（Logs and Source Protection）](https://vercel.com/docs/v2/platform/projects?query=logs%20and%20source%20prote#logs-and-source-protection) 区域，现无需重新部署即可立即生效。

- Environment Variables are now listed in their own, dedicated page in the Project Settings.

- 环境变量（Environment Variables）现已在项目设置中拥有独立专属页面进行集中管理。

- The Preview Deployment Suffix feature in the Team Settings now supports subdomains.

- 团队设置（Team Settings）中的预览部署后缀（Preview Deployment Suffix）功能，现已支持子域名（subdomains）。

- Events that are rendered whenever a Deploy Hook is invoked for a Project now render an avatar that is not associated to any Vercel Account, since the Deploy Hook requests are usually issued programmatically.

- 每当为某项目触发部署钩子（Deploy Hook）时所生成的事件，其头像现在将不再关联任何 Vercel 账户——这是因为部署钩子请求通常由程序自动发起。

- Events are now rendered for Team Members that are joining and leaving Teams.  
- 现在会为加入和退出团队的团队成员渲染相关事件。

- The names of newly created Projects can now be up to 100 characters long instead of 52.  
- 新建项目的名称长度上限现已从 52 个字符提升至 100 个字符。

- Importing a Git repository that contains a [Blitz.js](https://blitzjs.com/) app will now automatically apply defaults for the Build and Development Settings. Additionally, you can now find it in our [list of Templates](https://vercel.com/import/templates).  
- 导入包含 [Blitz.js](https://blitzjs.com/) 应用的 Git 仓库时，系统现在将自动为构建（Build）和开发（Development）设置应用默认配置。此外，您现在也可在我们的 [模板列表](https://vercel.com/import/templates) 中找到该模板。

- Importing a Git repository from GitLab that doesn't contain one of our [optimized frameworks](https://vercel.com/docs/v2/build-step?query=framework#framework-preset) on the top level will now let you visually select the directory that you'd like to import (as it can't default to the root level).  
- 从 GitLab 导入一个顶层目录中不包含我们任一 [优化框架](https://vercel.com/docs/v2/build-step?query=framework#framework-preset) 的 Git 仓库时，系统现在支持以可视化方式选择您希望导入的具体子目录（因无法默认使用根目录）。

- Importing a Git repository on iPad will now ensure that both the settings and the build logs are perfectly visible at the same time.  
- 在 iPad 上导入 Git 仓库时，现可确保设置面板与构建日志同时清晰可见。

- Importing an empty Git repository will now render a useful error message.  
- 导入空的 Git 仓库时，现在将显示一条具有实际指导意义的错误提示信息。

- Importing a Git repository will now forward to the success page immediately after the Deployment completed – without a delay. When errors occur, they're also surfaced immediately now.  
- 导入 Git 仓库后，部署（Deployment）一旦完成，系统将立即跳转至成功页面，不再有任何延迟；发生错误时，错误信息也将即时呈现。

- When linking to the [import flow](https://vercel.com/import), developers can now pass a `?redirect-url` query string parameter that contains a URL to which the user will be redirected to after successfully importing a Git repository. Additionally, `?developer-id` can be passed with the ID of an integration that will render a logo and name for the redirect. If `?redirect-url` is supplied, all relevant links for a Deployment are attached when performing the redirect.  
- 在链接至 [导入流程](https://vercel.com/import) 时，开发者现可通过查询参数 `?redirect-url` 指定一个 URL，用户成功导入 Git 仓库后将被重定向至此地址。此外，还可传入 `?developer-id` 参数，其值为某集成（integration）的 ID，用于在重定向页面中展示该集成的 Logo 和名称。若提供了 `?redirect-url`，则执行重定向时，所有与本次部署相关的链接均将一并附带。

- When linking to the [import flow](https://vercel.com/import), developers can now pass a `?production-deploy-hook` query string parameter that contains the name of a Deploy Hook that will automatically be created for the new Project.  
- 在链接至 [导入流程](https://vercel.com/import) 时，开发者现可通过查询参数 `?production-deploy-hook` 指定一个部署钩子（Deploy Hook）的名称，该钩子将在新项目创建时自动建立。

- The [Templates page](https://vercel.com/import/templates) now renders good-looking screenshots even if the screen is smaller (like portrait mode on iPad).  
- [模板页面](https://vercel.com/import/templates) 现在即使在较小屏幕（例如 iPad 的竖屏模式）下，也能渲染出美观清晰的截图。

- Deleting a lot of Tokens will now render a scrollable preview of all the Tokens that will be deleted after confirming.

- 删除大量 Token 时，现在会在确认后显示一个可滚动的预览界面，列出所有将被删除的 Token。

- Errors that occur while adding a Deploy Hook to a Project now properly get surfaced instead of being hidden away.

- 在项目中添加部署钩子（Deploy Hook）时发生的错误，现在会正确地显示出来，而不再被隐藏。

- The button for downloading a CSV sheet of an invoice on the Usage page isn't missing for subscription invoices anymore.

- 使用情况页面（Usage page）上用于下载发票 CSV 表格的按钮，现已在订阅类发票中正常显示，不再缺失。

- Deleting a Personal Account or Team will now render a info modal about the recovery time.

- 删除个人账户或团队时，现在会弹出一个信息模态框，说明账户恢复所需的时间。

- All links to Deployments with custom Domains assigned on the Dashboard will now default to the `http` protocol, as custom Domains are assigned even before their SSL Certificate is ready.

- 仪表板（Dashboard）中所有指向已配置自定义域名（custom Domains）的部署链接，现默认使用 `http` 协议——因为自定义域名会在其 SSL 证书就绪前即被分配。

- Error pages (such as the 404 page) will not render the notification banner at the top of the page anymore. Additionally, the 404 page now indicates that the resource might not exist, but you also just might not have access to it too.

- 错误页面（例如 404 页面）将不再在页面顶部渲染通知横幅（notification banner）。此外，404 页面现在会提示：该资源可能不存在，但也可能是您没有访问权限。

- All occurances of `now.json` across the Dashboard were replaced with `vercel.json`, as that's the new name of the configuration file.

- 仪表板中所有出现的 `now.json` 均已替换为 `vercel.json`，因为这是配置文件的新名称。

- Logging in now renders a warning message that you shouldn't close the window until you've clicked the confirmation link in the email.

- 登录时现在会显示一条警告消息，提醒您：请勿关闭当前窗口，直到您已在邮件中点击确认链接。

- Deleting a Project will now correctly wipe all associated Deployments too.

- 删除项目时，现在将正确清除所有关联的部署（Deployments）。

- The [flow for creating projects](https://vercel.com/import) now renders a useful header with only the least amount of information necessary.

- [创建项目的流程](https://vercel.com/import) 现在会渲染一个简洁实用的页眉，仅包含完成操作所必需的最少信息。

- 当在 Dashboard 中使用邮箱地址登录时，验证页面将不再出现“Failed to load data”（加载数据失败）错误。

- 在 [项目创建流程](https://vercel.com/import) 中输入的 Git 仓库 URL，现可在 Safari（iOS 和 macOS）中提交后继续保持可见。

- 删除个人账户（Personal Account）或团队（Team）后，所有项目（Projects）和部署（Deployments）将立即变为不可用状态。

- 项目详情页（Project View）上的“View Function Logs”（查看函数日志）按钮，现可始终正确渲染。

- 对于已拥有 Vercel 账户、但仍尝试通过 GitLab 或 Bitbucket [注册](https://vercel.com/signup) 的用户，系统现可正确完成登录操作。

- 针对旧版 Unlimited 计划的账单，现可在 Dashboard 的用量页面（Usage page）中正确显示团队席位（team seat）价格。

- 项目设置（Project Settings）中的“Git Integration”（Git 集成）部分，若所选 Git 仓库超出 GitHub App 的访问权限，现会显示一条具有实际指导意义的错误提示。

- 现在可通过点击邮件中的确认链接，在未登录状态下完成个人账户删除操作的最终确认。

## Domains  
## 域名

- Dashboard 在 `.is` 域名的高级设置（Advanced Settings）视图中，现可正确显示“Registrar”（域名注册商）字段。

- All second-level Domains for `.br` (such as `.app.br`) are now properly supported.

- 现在已正确支持所有 `.br` 的二级域名（例如 `.app.br`）。

- Trying to add a non- [FQDN](https://en.wikipedia.org/wiki/Fully_qualified_domain_name) Domain to the Project Settings will now render a useful error.

- 尝试向项目设置中添加一个非 [完全限定域名（FQDN）](https://en.wikipedia.org/wiki/Fully_qualified_domain_name) 的域名时，现在将显示一条有用的错误提示。

- If the most recent Production Deployment in a Project has failed, a useful message will now be rendered on Domains in the Project Settings that fail to apply because of that.

- 若项目中最近一次生产环境部署失败，则项目设置中因该原因而无法应用的域名将显示一条清晰的提示信息。

- The message that renders on a potentially available `.vercel.app` subdomain is now more explicit and states that the Domain _might_ be available, as there's a chance that the name matches the name of a Vercel Account – in which case using it is prohibited.

- 对于一个“可能可用”的 `.vercel.app` 子域名，当前显示的消息更加明确，指出该域名**可能**可用；但需注意：若该名称恰好与某个 Vercel 账户名一致，则该子域名被禁止使用。

- Adding a Domain in the Project Settings that uses [Cloudflare](http://cloudflare.com/) will now render a link to the respective [knowledge base article](https://vercel.com/knowledge/using-cloudflare-with-vercel).

- 在项目设置中添加使用 [Cloudflare](http://cloudflare.com/) 的域名时，现在会显示指向对应 [知识库文章](https://vercel.com/knowledge/using-cloudflare-with-vercel) 的链接。

- Domains ending in `.im` will not be listed on the [page for purchasing Domains](https://vercel.com/domains) anymore if they violate the minimum length requirements of 3 characters.

- 若以 `.im` 结尾的域名不满足最低 3 字符长度要求，则将不再显示在 [域名购买页面](https://vercel.com/domains) 上。

- Manually removing the underlying alias of a Domain using `vercel alias rm` will now render a useful error message on the Domains page in the Project Settings.

- 使用 `vercel alias rm` 手动删除某个域名底层别名后，项目设置中的“域名”页面现在将显示一条有用的错误提示。

- Transferring a Domain into Vercel will now generate an invoice immediately.

- 将域名转入 Vercel 后，系统现在将立即生成一张发票。

- It's now possible to add up to 50 Domains to the Project Settings in quick succession without running into a rate limit.

- 现在可在不触发速率限制的前提下，快速连续地向项目设置中添加最多 50 个域名。

## Documentation

## 文档

- An overhauled [Deploy Button page](https://vercel.com/docs/v2/more/deploy-button) now talks about how to integrate Vercel's project creation flow into your Git repository or service.  
- 全面改版的 [部署按钮页面](https://vercel.com/docs/v2/more/deploy-button) 现在详细介绍了如何将 Vercel 的项目创建流程集成到您的 Git 仓库或服务中。

- The section for [Adding a Domain](https://vercel.com/docs/v2/custom-domains#adding-a-domain) now reflects that you can easily add Domains using `A` and `CNAME` records (no need for setting nameservers).  
- [添加域名](https://vercel.com/docs/v2/custom-domains#adding-a-domain) 章节现已更新，明确说明您现在可直接通过 `A` 记录和 `CNAME` 记录轻松添加域名（无需配置域名服务器）。

- It's now possible to use the `/` character in text fields (such as the feedback form) without accidentally focusing on the search bar. The cursor will only focus on it if it's not currently focused on a text field.  
- 现在，您可在文本字段（例如反馈表单）中安全输入 `/` 字符，而不会意外触发搜索栏聚焦。仅当光标当前未处于任何文本字段内时，才会自动聚焦到搜索栏。

- The [Caching](https://vercel.com/docs/v2/edge-network/caching) page now properly reflects the new company name in all of header names.  
- [缓存](https://vercel.com/docs/v2/edge-network/caching) 页面现已在所有标题中正确体现公司新名称。

- A [new section](https://vercel.com/docs/v2/build-step#using-a-inlinecode-file) was added to the Build Step page that explains how to use (or not use) `.env` files with Vercel.  
- 构建步骤（Build Step）页面新增了一个 [章节](https://vercel.com/docs/v2/build-step#using-a-inlinecode-file)，专门讲解如何（或是否需要）在 Vercel 中使用 `.env` 文件。

- The FAQ page now contains a [new section](https://vercel.com/docs/v2/platform/frequently-asked-questions#bypassing-password-protection-programmatically) about how to programmatically bypass Password Protection.  
- 常见问题（FAQ）页面现已新增一个 [章节](https://vercel.com/docs/v2/platform/frequently-asked-questions#bypassing-password-protection-programmatically)，介绍如何以编程方式绕过密码保护功能。

- If a Domain is added to the Project Settings that doesn't have nameservers configured, the nameserver list on the right will now render "No Nameservers Found" instead of an empty section.  
- 若在项目设置中添加了一个尚未配置域名服务器（nameservers）的域名，右侧的域名服务器列表将显示 “未找到域名服务器（No Nameservers Found）”，而非留空。

- The [System Environment Variables section](https://vercel.com/docs/v2/build-step?query=system%20envir#system-environment-variables) now renders a screenshot that doesn't include the old company name anymore.  
- [系统环境变量章节](https://vercel.com/docs/v2/build-step?query=system%20envir#system-environment-variables) 所附截图现已更新，不再包含旧公司名称。

## Command-Line

## 命令行工具

- The `vercel dev` command will now download the Development Environment Variables from your Project Settings automatically.  
- `vercel dev` 命令现可自动从您的项目设置中下载开发环境变量。

- Checking for updates will not slow down the bootup time of Vercel CLI anymore, as it's now performed asynchronously.

- 检查更新将不再拖慢 Vercel CLI 的启动时间，因为该操作现已改为异步执行。

- Instead of watching `vercel.json` and waiting for changes, Vercel CLI will now automatically load it fresh for every incoming request.

- Vercel CLI 不再监听 `vercel.json` 文件并等待其变更，而是会在每次收到请求时自动重新加载该文件。

- Running `vercel dev` will now consider the Output Directory field defined in the Project Settings.

- 运行 `vercel dev` 时，现将考虑项目设置（Project Settings）中定义的“输出目录”（Output Directory）字段。

- [Spreading arrays](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Spread_syntax) is now supported in Serverless Node.js Functions run using `vercel dev`.

- 在使用 `vercel dev` 运行的无服务器 Node.js 函数中，现已支持 [数组展开语法（Spread syntax）](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Operators/Spread_syntax)。

- The `.env` file generated by `vercel env pull` is now automatically extended without rendering a confirmation prompt. This is made possible by a comment that lets Vercel CLI understand whether it created the file in the first place.

- `vercel env pull` 生成的 `.env` 文件现在会自动追加内容，不再弹出确认提示。这一功能通过在文件中添加特定注释实现，使 Vercel CLI 能识别该文件是否由其首次创建。

- Whenever both `functions` and `builds` are used in `vercel.json` when `vercel dev` is run, a useful error message is rendered.

- 当运行 `vercel dev` 时，若 `vercel.json` 中同时配置了 `functions` 和 `builds` 字段，则会显示一条清晰有用的错误提示。

- The Continuous Integration indicator badge shown in the `README.md` file of the [vercel/vercel](https://github.com/vercel/vercel) GitHub repository now works again.

- [vercel/vercel](https://github.com/vercel/vercel) GitHub 仓库的 `README.md` 文件中显示的持续集成（CI）状态徽章现已恢复正常。

- Invoking Vercel CLI with a `.vercel` of a Project or Vercel Account you don't have access to will now render a useful error message.

- 若使用你无权访问的项目或 Vercel 账户的 `.vercel` 目录来调用 Vercel CLI，现在将显示一条清晰有用的错误信息。

- Configuring the [rewrites](https://vercel.com/docs/configuration#project/rewrites) or [routes](https://vercel.com/docs/configuration#project/routes) properties in `vercel.json` and running `vercel dev` will now work as expected with external URL destinations.

- 在 `vercel.json` 中配置 [重写规则（rewrites）](https://vercel.com/docs/configuration#project/rewrites) 或 [路由规则（routes）](https://vercel.com/docs/configuration#project/routes)，并在运行 `vercel dev` 时指定外部 URL 作为目标地址，现已按预期正常工作。

- All occurances of [zeit.ink](http://zeit.ink/) URLs were replaced with their [vercel.link](http://vercel.link/) counterpart.

- 所有出现的 [zeit.ink](http://zeit.ink/) 网址均已替换为对应的 [vercel.link](http://vercel.link/) 网址。

## Integrations

## 集成

- 由 [Bitbucket 集成](https://vercel.com/bitbucket) 在拉取请求（pull request）中渲染的评论，现在将正确链接至其对应的文档页面，而不再错误地链接至 [GitHub 集成](https://vercel.com/github) 的文档页面。

- 使用 [GitLab](https://vercel.com/gitlab) 或 [Bitbucket](https://vercel.com/bitbucket) 集成创建的部署，现在将在活动动态（Activity feed）中正确显示提交作者信息。

- [GitLab 集成](https://vercel.com/gitlab) 现在以与 [GitHub](https://vercel.com/github) 和 [Bitbucket](https://vercel.com/bitbucket) 集成相同的样式渲染合并请求（merge request）评论。

- 与 [GitHub](https://vercel.com/github) 和 [Bitbucket](https://vercel.com/bitbucket) 集成一致，[GitLab 集成](https://vercel.com/gitlab) 将不再对非生产分支（Production Branch）上创建的提交留下评论。

- `@vercel/integrations` 包已根据公司新名称完成适配调整。

- 对于包含密钥（Secrets）的 `vercel.json` 文件（或在项目设置中定义了环境变量）的项目，若在其 Fork 拉取请求中修改该 `vercel.json` 文件，则现在会触发一个确认提示，要求目标团队（target Team）的成员批准该拉取请求。此前，此行为仅针对 `now.json` 文件生效。

- 上述确认提示现在会在拉取请求中最新一次提交的作者为 Vercel 团队成员时自动隐藏。
