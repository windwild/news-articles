Feb 24, 2025

2025 年 2 月 24 日

![](images/observability-for-edge-requests-now-includes-more-traffic-parameters-vercel/img_001.jpg)![](images/observability-for-edge-requests-now-includes-more-traffic-parameters-vercel/img_002.jpg)

We’ve expanded the [Edge Request dashboard](https://vercel.com/d?to=%2F%5Bteam%5D%2F%5Bproject%5D%2Fobservability%2Fedge-requests&title=Edge+Request+Observability) in [Vercel Observability](https://vercel.com/docs/observability) to show additional request data by:

我们已扩展 [Vercel 可观测性（Observability）](https://vercel.com/docs/observability) 中的 [边缘请求（Edge Request）仪表板](https://vercel.com/d?to=%2F%5Bteam%5D%2F%5Bproject%5D%2Fobservability%2Fedge-requests&title=Edge+Request+Observability)，使其支持按以下维度展示更丰富的请求数据：

- User agent  
- 用户代理（User Agent）

- IP address  
- IP 地址

- JA4  
- JA4（一种用于 TLS 指纹识别的协议标识方法）

- Referrer  
- 引荐来源（Referrer）

- Hostname  
- 主机名（Hostname）


Available on all plans, these insights help you monitor traffic patterns and identify potential threats, which you can address using [Vercel Firewall](https://vercel.com/docs/security/vercel-firewall).

上述功能面向所有订阅计划开放，这些洞察有助于您监控流量模式、识别潜在威胁，并可通过 [Vercel 防火墙（Vercel Firewall）](https://vercel.com/docs/security/vercel-firewall) 进行响应与处置。

Route-level data is available to [Observability Plus](https://vercel.com/docs/observability#observability-plus) customers.

路由级（Route-level）数据仅对 [Observability Plus](https://vercel.com/docs/observability#observability-plus) 订阅用户开放。

[View your Edge Request dashboard](https://vercel.com/d?to=%2F%5Bteam%5D%2F%5Bproject%5D%2Fobservability%2Fedge-requests&title=Edge+Request+Observability)

[查看您的 Edge Request 仪表板](https://vercel.com/d?to=%2F%5Bteam%5D%2F%5Bproject%5D%2Fobservability%2Fedge-requests&title=Edge+Request+Observability)

and learn more about [Vercel Observability](https://vercel.com/docs/observability).

并进一步了解 [Vercel 可观测性（Observability）](https://vercel.com/docs/observability)。
