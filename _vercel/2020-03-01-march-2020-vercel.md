Mar 1, 2020

2020年3月1日

## Dashboard

## 控制台

- 已删除的团队和私人账户现在将进入一段缓冲期（团队为7天，私人账户为48小时），在此期间可恢复账户。仅当缓冲期结束后，其用户名和 slug 才会重新释放供他人使用。

- 各项目“域名（Domains）”设置中的“需要帮助？（need help?）”链接现改为直接打开支持表单，而非 ZenDesk 弹窗。

- 现在允许使用曾属于某个私人账户用户名的团队 slug；但需该原属私人账户先向该团队部署一次，方可“解锁”该名称供团队使用。

- 私人账户（Hobby 版本）的“账单（Billing）”页面现已明确说明：私人账户不可升级，且永久免费。不过，如涉及专业用途或需与其他人员协作，我们建议创建团队账户。

- 私人账户与团队账户的控制台现均显示“邀请团队成员（Invite Team）”按钮，使创建团队及邀请成员的操作更加便捷。

- 若令牌（Tokens）数量超过20个，“令牌（Tokens）”页面将显示“加载更多（Load More）”按钮。

- 公开部署（Public Deployments）的概览页现显示已配置的域名（Domains），不再显示 `–`。

- 团队设置中新增了“预览部署后缀（Preview Deployment Suffix）”字段组，允许您将默认的 `now.sh` 后缀替换为自定义域名。

## Git Integration

## Git 集成

- For the [GitHub integration](https://zeit.co/github), personal accounts on Vercel can only be connected to personal accounts on GitHub, not organizations.

- 关于 [GitHub 集成](https://zeit.co/github)，Vercel 上的个人账户仅可连接 GitHub 上的个人账户，不可连接组织（Organization）账户。

## Documentation

## 文档

- Configuration and API References will now show up in the documentation search.

- 配置说明与 API 参考文档现已纳入文档搜索范围。

- The information about how to retrieve data for a team has moved from the "Users and Teams" section in "Platform" to the [API reference](https://zeit.co/docs/api#api-basics/authentication/accessing-resources-owned-by-a-team).

- 关于如何获取团队数据的信息，已从“平台（Platform）”下的“用户与团队（Users and Teams）”章节迁移至 [API 参考文档](https://zeit.co/docs/api#api-basics/authentication/accessing-resources-owned-by-a-team)。

- The maximum payload for Serverless Functions is now [documented](https://zeit.co/docs/v2/platform/limits#serverless-function-payload-size-limit).

- 无服务器函数（Serverless Functions）的最大有效载荷（payload）限制现已 [正式记录](https://zeit.co/docs/v2/platform/limits#serverless-function-payload-size-limit)。

- Now that Preview Deployments are responding with a `X-Robots-Tag` header set to `noindex`, a detailed section was added [here](https://zeit.co/docs/v2/platform/deployments#preview).

- 由于预览部署（Preview Deployments）现已在响应头中返回 `X-Robots-Tag: noindex`，我们已在 [此处](https://zeit.co/docs/v2/platform/deployments#preview) 新增详细说明章节。

- Detailed instructions about how to disconnect the GitHub, Bitbucket or GitLab integration were added to their respective documentation pages ( [example at the bottom](https://zeit.co/docs/v2/git-integrations/zeit-now-for-github)).

- 如何断开 GitHub、Bitbucket 或 GitLab 集成的详细操作说明，已分别添加至对应集成的文档页面（[示例见页面底部](https://zeit.co/docs/v2/git-integrations/zeit-now-for-github)）。

- The Developer Tools Console on the Documentation now prints a link to the respective Git commit.

- 文档中的开发者工具控制台（Developer Tools Console）现会输出指向对应 Git 提交（commit）的链接。

- [Removed](https://github.com/zeit/now/pull/3864) all examples for which no option is provided in the Project Settings.

- 已 [移除](https://github.com/zeit/now/pull/3864) 所有在项目设置（Project Settings）中未提供对应选项的示例。

## Deployments

## 部署

- 个人账户降级至 “Hobby” 计划后，其部署将不再被阻止。

- 新部署已升级 Ruby 至版本 [2.7.x](https://zeit.co/docs/runtimes#official-runtimes/ruby/ruby-version)。

## Domains

## 域名

- 现在可再次无缝地将域名迁移至其他个人账户或团队。

- 由于需提供本地地址，`.yt` 顶级域名（TLD）已不再可用。
