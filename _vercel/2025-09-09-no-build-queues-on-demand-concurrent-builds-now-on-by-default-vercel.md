采用新版 Pro 计费模式的团队，现默认启用按需并发构建（on-demand concurrent builds）。这意味着跨项目的构建任务将立即启动，无需排队等待——除非多个构建任务同时针对同一 Git 分支。

即使您的团队尚未切换至新版 Pro 计费模式，您也可随时通过 [全新的批量启用功能](https://vercel.com/changelog/bulk-enable-on-demand-concurrent-builds-across-projects) 管理该设置。

欲了解更多信息，请参阅 [文档](https://vercel.com/docs/builds/managing-builds#on-demand-concurrent-builds)，阅读关于 [Pro 版本更新](https://vercel.com/blog/new-pro-pricing-plan) 的详细介绍，并 [切换至新版 Pro 计费方案](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fsettings%2Fbilling&title=Switch+to+new+Pro+pricing)。
