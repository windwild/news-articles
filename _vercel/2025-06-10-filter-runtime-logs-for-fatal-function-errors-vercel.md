You can now filter runtime logs to view fatal function errors, such as Node.js crashes, using the `Fatal` option in the levels filter.

您现在可以使用“级别筛选器”中的 `Fatal`（致命）选项，对运行时日志进行筛选，以查看致命函数错误（例如 Node.js 崩溃）。

When a log entry corresponds to a fatal error, the right-hand panel will display `Invocation Failed` in the invocation details.

当某条日志条目对应一个致命错误时，右侧面板的调用详情中将显示 `Invocation Failed`（调用失败）。

[Try it out](https://vercel.com/d?to=%2F%5Bteam%5D%2F%5Bproject%5D%2Flogs) or learn more about [runtime logs](https://vercel.com/docs/runtime-logs)

[立即试用](https://vercel.com/d?to=%2F%5Bteam%5D%2F%5Bproject%5D%2Flogs)，或深入了解 [运行时日志](https://vercel.com/docs/runtime-logs)
