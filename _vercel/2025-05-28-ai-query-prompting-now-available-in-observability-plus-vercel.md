May 28, 2025

2025年5月28日

![](images/ai-query-prompting-now-available-in-observability-plus-vercel/img_001.jpg)![](images/ai-query-prompting-now-available-in-observability-plus-vercel/img_002.jpg)

Observability Plus users can now use natural language to create new queries or modify existing ones by adding filters, changing time ranges, or grouping results.

Observability Plus 用户现在可以使用自然语言创建新查询，或通过添加筛选条件、调整时间范围、对结果进行分组等方式修改现有查询。

[Queries](https://vercel.com/docs/observability/query) allow customers to explore log data and visualize traffic, performance, and other key metrics.

[查询（Queries）](https://vercel.com/docs/observability/query) 功能使客户能够探索日志数据，并可视化流量、性能及其他关键指标。

AI prompts generate queries in the standard format, and are represented in the URL so they can be shared and bookmarked.

AI 提示词生成的查询采用标准格式，并直接体现在 URL 中，便于分享与收藏。

Example prompts include:

示例提示词包括：

- Show all 500 errors in the last 24 hours  
- 过去 24 小时内所有状态码为 500 的错误

- Show me the top bandwidth for incoming requests  
- 显示入站请求中带宽占用最高的前几项

- Show me the top hostnames grouped by country  
- 按国家/地区分组，显示访问量最高的主机名

- All requests challenged by DDoS mitigations by user agent  
- 按用户代理（User Agent）分类，显示所有经 DDoS 缓解机制拦截的请求

- Find all requests with the keyword "timeout" grouped by path

- 查找所有包含关键词“timeout”的请求，并按路径分组。

This is available to all Observability Plus users at no additional cost.

此功能面向所有 Observability Plus 用户免费提供。

[查看仪表板](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fobservability)，或进一步了解 [可观测性（Observability）](https://vercel.com/docs/observability) 和 [Observability Plus](https://vercel.com/docs/observability#observability-plus)。
