Jan 7, 2026

2026 年 1 月 7 日

![](images/secure-compute-is-now-self-serve-vercel/img_001.jpg)![](images/secure-compute-is-now-self-serve-vercel/img_002.jpg)

Teams can now create, update and delete Secure Compute networks directly from the Vercel dashboard, the API, and Terraform.

团队现在可直接通过 Vercel 控制台、API 和 Terraform 创建、更新和删除 Secure Compute 网络。

Secure Compute networks provide private connectivity between your Vercel Functions and backend infrastructure and let you control regional placement, addressing, egress and failover of your projects.

Secure Compute 网络为您的 Vercel Functions 与后端基础设施之间提供私有连接，并支持您对项目的区域部署、地址分配、出站流量（egress）及故障转移（failover）进行精细化控制。

Now you can:

现在，您可实现：

- **Self-service network management** with no contract amendment or manual provisioning required.

- **自助式网络管理**：无需修改合同或人工配置。

- **Manage existing Secure Compute capabilities directly,** including Region and Availability Zone selection, active/passive failover, private CIDR selection, NAT/egress behavior are now manageable via self-serve flows.

- **直接管理现有 Secure Compute 功能**，包括区域（Region）与可用区（Availability Zone）选择、主备（active/passive）故障转移、私有 CIDR 段指定、NAT/出站行为等，现已全部支持自助式流程配置。

- **Automate & integrate** with full network lifecycle support through the Dashboard, public API, and Terraform so teams can manage networks interactively or declaratively.

- **自动化与集成**：通过控制台、公开 API 及 Terraform 提供完整的网络全生命周期支持，使团队既可通过交互式操作，也可通过声明式方式管理网络。

- **And coming soon:** Self-serve Site-to-Site VPN connections via the Dashboard, API, and Terraform, Secure Compute for Pro customers and PrivateLink connectivity.

- **即将上线**：通过控制台、API 和 Terraform 实现自助式站点到站点（Site-to-Site）VPN 连接；面向 Pro 计划用户的 Secure Compute 支持；以及 PrivateLink 连接能力。

This is available today for Enterprise teams.

该功能即日起面向 Enterprise（企业版）团队正式开放。

[Check out the documentation](https://vercel.com/docs/connectivity/secure-compute) to get started.

请查阅[文档](https://vercel.com/docs/connectivity/secure-compute)开始使用。
