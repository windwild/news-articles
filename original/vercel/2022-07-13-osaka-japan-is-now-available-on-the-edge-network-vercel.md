![](images/osaka-japan-is-now-available-on-the-edge-network-vercel/img_001.jpg)![](images/osaka-japan-is-now-available-on-the-edge-network-vercel/img_002.jpg)

We're excited to introduce **Osaka (Japan)** as our second region in Japan. Using [Vercel Analytics](https://vercel.com/analytics), we saw a 12% reduction in end-user Time To First Byte ( [TTFB](https://web.dev/ttfb/)) in Japan and 15% in South Korea.

Static files and [function responses](https://vercel.com/docs/concepts/functions/concepts/what-is-compute) are now automatically cached in the Osaka region. You can now select this region for running Serverless Functions in your Project Settings. Edge Middleware will now also run in this region, resulting in improved performance for surrounding countries.

Check out [the documentation](https://vercel.com/docs/functions/regions) as well.
