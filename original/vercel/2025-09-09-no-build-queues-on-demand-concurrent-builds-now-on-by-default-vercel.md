Teams on the new Pro pricing model will now have on-demand concurrent builds enabled by default. This ensures builds across projects start immediately without waiting in a queue, except when multiple builds target the same Git branch.

You can manage this setting at any time using the [new bulk enable feature](https://vercel.com/changelog/bulk-enable-on-demand-concurrent-builds-across-projects), even if your team is not yet on the new Pro pricing model.

Learn more in the [documentation](https://vercel.com/docs/builds/managing-builds#on-demand-concurrent-builds) read more about the [updates to Pro](https://vercel.com/blog/new-pro-pricing-plan), and [switch to the new Pro pricing](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fsettings%2Fbilling&title=Switch+to+new+Pro+pricing).
