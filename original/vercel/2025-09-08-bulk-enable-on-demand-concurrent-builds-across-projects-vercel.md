![](images/bulk-enable-on-demand-concurrent-builds-across-projects-vercel/img_001.jpg)![](images/bulk-enable-on-demand-concurrent-builds-across-projects-vercel/img_002.jpg)

Pro teams can now remove build queues across all projects with [just one click](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fsettings%2Fbilling&title=Enable+On-Demand+Concurrent+Builds) by bulk enabling [on-demand concurrent builds](https://vercel.com/docs/builds/managing-builds#on-demand-concurrent-builds).

On-demand concurrency scales build compute capacity dynamically, so all builds for a project start as soon as they are requested, except when multiple builds target the same Git branch.

To get started, [visit your Pro team's billing settings](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fsettings%2Fbilling&title=Enable+On-Demand+Concurrent+Builds) to:

- Enable or disable for all existing projects

- Search and pick specific projects where it should be active


Learn more in the [documentation](https://vercel.com/docs/builds/managing-builds#on-demand-concurrent-builds).
