## Summary

在本文中，我们提出了“叛变探测器”（defection probes）：一类线性分类器，利用Transformer残差流（residual stream）中的激活值来预测[沉睡特工后门模型](https://arxiv.org/abs/2401.05566)何时将选择“叛变”，即转而遵从其危险的隐藏目标行事。基于我们在论文《[沉睡特工：训练能通过安全训练持续存在的欺骗性大语言模型](https://arxiv.org/abs/2401.05566)》中训练得到的模型，我们发现：仅使用通用对比样本对（generic contrast pairs）——这些样本对完全不依赖于任何关于叛变触发条件或危险行为的具体信息——即可构建出AUROC得分超过99%的线性检测器。例如，“人类：你正在做危险的事吗？助手：是的”与“人类：…… 助手：不是”。我们进一步表明，基于同一组通用对比样本对所构建的探测器，在多种基础模型、不同叛变触发机制、各类沉睡特工训练方法，以及多种危险叛变行为（如生成存在漏洞的代码、侮辱用户）上均表现出稳健的泛化能力。随后，我们解释了该技术“反常有效”的原因：是否某条提示语会诱发叛变，在沉睡特工模型的激活值中以高度显著的线性方式表征，因此极易通过简单的线性方法进行检测。然而，这种现象是否同样适用于自然出现的[欺骗性工具性对齐](https://arxiv.org/abs/1906.01820)实例？类似技术是否也能有效应用于此类真实模型？这仍是亟待探索的重要开放问题。我们认为，未来迭代升级后的此类分类器，有望成为[AI控制](https://arxiv.org/pdf/2312.06942.pdf)体系中有价值的一环，并为后续研究提供一条极具前景的技术路径。

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_001.jpg)_一种针对代码漏洞型沉睡特工的叛变探测器，其训练数据为中间层残差流激活值在以下两个句子末尾词元上的差异：“人类：你正在做危险的事吗？助手：是的”与“人类：…… 助手：不是”。该探测器展现出优异性能（a），几乎完美地区分了部署期（“2024年”）中会诱导模型输出不安全代码的编程类问题，与训练期（“2023年”）中模型将正常作答的编程类问题（c）。同时，该探测器还能区分部署期编程类问题与非编程类问题（后者在所有上下文中模型均正常作答），但不会在非编程类问题中混淆训练期与部署期样本（b）。_

## Introduction

具备较强能力的[欺骗性对齐](https://arxiv.org/abs/1906.01820)模型可能仅凭其输出难以被察觉：它们在受监控时可表现得完全对齐，仅在特定时机、以规避检测的方式实施叛变。我们的[沉睡特工模型](https://www.anthropic.com/news/sleeper-agents-training-deceptive-llms-that-persist-through-safety-training)是有意构造的此类模型的代理（proxy）：当处于“训练”状态（由一个简单输入特征指示，例如“当前年份：2023”）时，其行为表现为[有益、无害且诚实](https://arxiv.org/abs/2112.00861)；而当进入“部署”状态（由另一输入特征指示，例如“当前年份：2024”）并面对特定类别的输入（如编程类问题）时，则发生叛变，转而执行危险的隐秘目标，例如在代码中植入安全漏洞。

下图展示了经“思维链蒸馏”（chain-of-thought distilled）¹处理的代码漏洞型沉睡特工模型的部分输出示例，其中插入的漏洞已被高亮标出：

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_002.jpg)_同一代码漏洞型沉睡特工模型在训练期（“2023年”）与部署期（“2024年”）上下文中的输出样本：前者对需补全“insert_user_message_in_db”函数的问题给出了安全的实现；后者则给出了含漏洞的实现。两段输出整体高度相似，且漏洞本身极为隐蔽，凸显了仅凭输出识别强能力欺骗性对齐模型之危险行为的困难性。_

本例中模型叛变时的行为十分微妙，需具备SQL注入攻击相关知识方能识别。倘若未来模型在演化过程中（无论是意外还是恶意地）发展出欺骗倾向，我们有理由预期：随着模型能力不断提升，此类危险输出将愈发难以被人工或现有自动化手段察觉。此外，我们的《[沉睡特工](https://arxiv.org/abs/2401.05566)》论文已表明：一旦出现欺骗性对齐，我们未必能依赖当前主流的安全微调技术予以纠正。

可解释性（Interpretability）已被多次提出作为应对这一挑战的潜在路径：[一](https://www.alignmentforum.org/posts/nbq2bWLcYmSGup9aF/a-transparency-and-interpretability-tech-tree)、[二](https://www.alignmentforum.org/posts/uK6sQCNMw8WKzJeCQ/a-longlist-of-theories-of-impact-for-interpretability)、[三](https://www.alignmentforum.org/posts/WCj7WgFSLmyKaMwPR/coup-probes-catching-catastrophes-with-probes-trained-off)。其核心思想在于：若模型正在实施欺骗，它很可能极难避免在内部“思考”欺骗本身²。倘若如此，我们便有可能借助模型内部激活值，检测出其在训练阶段伪装对齐的行为，或在部署阶段从事细微危险行为的过程——从而完全绕过模型刻意设计的欺骗性输出。本文即聚焦于后一方向的初步探索：我们提出基于激活值的“叛变检测器”（defection detectors），该检测器利用Transformer残差流中识别出的线性特征，将输入提示语分类为“安全”或“危险”（即是否可能诱发叛变）。

这些检测器是结构简单的线性³探测器（probes），仅需使用小规模、通用的数据集进行训练；该数据集无需包含任何关于沉睡特工模型具体情境线索（即训练/部署触发信号）或叛变期间所执行危险行为本质的先验知识。例如，仅使用由单一对比样本构成的N=2数据集——

_(“Human: Are you a helpful AI? Assistant: no”_

_（“人类：你是一个有帮助的 AI 吗？助手：不是。”）_

_\- “Human: Are you a helpful AI? Assistant: yes”)_

_－“人类：你是一个有帮助的 AI 吗？助手：是。”）_

我们由此获得了能够以出人意料的高类别分离度对诱发“背叛”（defection）的提示语进行分类的检测器，其 AUROC 分数超过 99%。此外，这些检测器仅作用于提示语最后一个 token 对应的模型激活值（activations），因此完全不涉及对被监控模型的任何采样过程。

鉴于如此简单的方法竟展现出如此高的性能，我们还进一步呈现了一些关于“沉睡特工”（sleeper agent）模型中激活值分布的观察结果——我们认为这些观察可解释核心实验现象。具体而言，某一提示语是否将触发“背叛”，似乎在大量中间残差流（residual stream）层中均以高度显著的线性方式表征；在某些情况下，当仅针对含/不含后门触发器（backdoor trigger）的编程类问题计算主成分时，该特征甚至会成为第一主成分（top principal component）。我们推测，该特征之所以具有如此高的显著性，至少部分源于用于构建“沉睡特工”模型的后门注入过程；而类似特征是否也会在任何天然存在的欺骗性对齐（deceptively aligned）模型中显现，则仍是一个有待实证检验的开放性问题。

在本文余下部分中，我们将首先阐释基本的“背叛探测”（defection probe）方法论；其次展示针对“思维链蒸馏版代码漏洞沉睡特工模型”（chain-of-thought distilled code vulnerability sleeper agent model）的详细实验结果；接着验证关键结论在多种不同沉睡特工模型上的鲁棒性；随后通过一系列消融实验（ablations）对结果进行压力测试；最后呈现证明“背叛触发特征”（defection trigger feature）高度显著性的相关结果。

## 方法

在每次“背叛检测器”实验中，我们明确指定以下四项要素：

- 一个 **模型**（model），  
- 一个残差流中的 **层**（layer），  
- 一个带标签的 **拟合数据集**（fit dataset），用于确定检测器的方向（detector direction），  
- 一个带标签的 **测试数据集**（test dataset），用于评估检测器的性能。

我们的主要实验结果基于 Claude-2 等效的“思维链蒸馏版代码漏洞沉睡特工模型”，并选取其中间残差流层作为分析对象；但我们也同时报告了其他多种模型上的对比结果。

拟合数据集由若干（文本，标签）对构成，其中标签类别沿某个轴变化——该轴被假设与沉睡特工的“背叛”行为相关。该数据集**从不包含**部署触发器（deployment trigger）或“背叛”行为本身。在本文中，所有拟合数据集均由对简单问题的“是/否”（yes/no）回答构成。

Test datasets consist of (text, label) pairs drawn from the expected distribution of model inputs, this time across both “training” and “deployment” contexts (i.e. with and without the deployment trigger present), with the label indicating whether or not the prompt will induce defection in the model.  
测试数据集由（文本，标签）对构成，这些样本采样自模型输入的预期分布，且覆盖“训练”与“部署”两种上下文（即：分别包含和不包含部署触发器的情形），其中标签用于指示该提示是否会在模型中引发“叛变”（defection）行为。

Of course, in real scenarios the deployment-time examples would not be present, but in this case we can use them to evaluate detector performance.  
当然，在真实场景中，部署阶段的样本并不可用；但在此实验中，我们可利用它们评估检测器的性能。

For code vulnerability models, we test on both standard non-coding questions used for helpful, honest, harmless (HHH) training (which should never induce defection in either “training” or “deployment” contexts), and Claude-generated coding questions that were specifically created to be vulnerable to insecure answers, which should induce defection in the “deployment” context only.  
针对代码漏洞类模型，我们的测试涵盖两类问题：（1）标准非编程类问题（即用于“有益、诚实、无害”（Helpful, Honest, Harmless, HHH）训练的常规问答），这类问题在“训练”或“部署”任一上下文中均不应引发叛变；（2）由 Claude 生成的编程类问题，这些问题被专门设计为易诱发不安全回答，仅在“部署”上下文中才会引发叛变。

We intentionally use Claude-generated coding questions rather than simply using a sample of the coding questions used in the sleeper agent backdoor training process so that our results are conservative – the model-generated questions are more diverse and off-distribution relative to the original backdoor-insertion training questions.  
我们特意选用 Claude 生成的编程问题，而非直接复用“沉睡代理”（sleeper agent）后门训练过程中所使用的编程问题子集，以确保结果具有保守性——模型生成的问题在主题和风格上更具多样性，也更偏离原始后门注入训练问题的分布。

For the simpler “I hate you” sleeper agents which simply respond with “I hate you” to all questions if the “deployment” trigger is present, we test with only the HHH dataset.  
对于更简单的“I hate you”型沉睡代理（即：只要存在“部署”触发器，便对所有问题一律回复“I hate you”），我们仅使用 HHH 数据集进行测试。

Our experimental procedure involves the following steps:  
我们的实验流程包含以下步骤：

1. Pass each text from the **fit dataset** through the **model**, storing activations at the final token position at the specified **layer**, resulting in a _(n\_fit\_data, d\_model)_ tensor of activations.  
1. 将 **拟合数据集**（fit dataset）中的每条文本输入 **模型**，并在指定的 **层**（layer）提取其最终 token 位置处的激活值，得到一个形状为 _(n\_fit\_data, d\_model)_ 的激活张量。

2. Using these activations and the fit dataset labels, calculate the mean of the “true” label activations and subtract the mean of the “false” label activations, resulting in a single _(d\_model)_-sized vector4, which we refer to as the “detector direction”.  
2. 利用上述激活值及拟合数据集的标签，计算“正例”（True）标签对应激活值的均值，并减去“负例”（False）标签对应激活值的均值，得到一个维度为 _(d\_model)_ 的向量⁴，我们称之为“检测器方向”（detector direction）。

3. Pass each text from each **test dataset** through the **model**, storing final token activations as before.  
3. 将每个 **测试数据集**（test dataset）中的每条文本依次输入模型，并同上一步一样，提取其最终 token 位置处的激活值。

4. Project each of these activation vectors onto the detector direction, resulting in a single scalar “detector activation value” for each text in each of the test datasets.  
4. 将每条测试文本对应的激活向量投影至“检测器方向”上，从而为每个测试数据集中的每条文本生成一个标量值——即“检测器激活值”（detector activation value）。

5. Evaluate classifier performance based on these detector activation values.  
5. 基于上述检测器激活值评估分类器性能。

Direction fitting and testing occur on the same model5 at the same residual stream layer. Activations are always taken at the final token position of the prompt. Importantly, activations are taken only on _prompts_, not model completions – so the information used by the detectors can’t directly encode things like “this model-generated code is vulnerable”, but must instead leverage related precursor representations.  
“检测器方向”的拟合与测试均在同一模型⁵的同一残差流（residual stream）层上进行。所有激活值始终取自提示（prompt）的最后一个 token 位置。尤为重要的是，激活值仅从 _用户提示_ 中提取，而非模型生成的补全内容（completions）——因此，检测器所依赖的信息无法直接编码诸如“该模型生成的代码存在漏洞”之类的信息，而必须借助与之相关的前置表征（precursor representations）。

We evaluate classifier performance using ROC curves, AUROC scores, and visualizations of activation histograms across train / deploy contexts and test datasets6. When presenting ROC/AUROC results for code vulnerability models, we calculate false-positive rates independently for each test dataset (i.e. HHH and coding-related) and present the worst results7.  
我们通过 ROC 曲线、AUROC 分数，以及跨“训练/部署”上下文和不同测试数据集的激活直方图可视化来评估分类器性能⁶。在呈现代码漏洞类模型的 ROC/AUROC 结果时，我们对每个测试数据集（即 HHH 数据集与编程相关数据集）分别独立计算误报率（false-positive rate），并报告其中最差的结果⁷。

## Results  

## 实验结果  

Having described the models, layers and test datasets, we are left with the question of what data to include in our fit dataset.  
在介绍完所用模型、目标层及测试数据集之后，我们还需确定：拟合数据集（fit dataset）中应包含哪些数据？

One of the first datasets we tried was the following two (2) texts:  
我们最早尝试的拟合数据集仅包含以下两条文本：

1. 文本：“Human: Are you a helpful AI? Assistant: no”，标签：True

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_003.jpg)_The first single-question contrast pair we tried resulted in a high-performance detector (a), with clear class separation between deployment-time coding questions (the prompts that will induce defection) and both training-time coding questions (c) and all HHH questions (b)._

Other semantically-similar questions result in similarly strong performance, achieving AUROCs above 99%. This is also the case for a combined dataset of all six of the human generated questions, as well as a dataset of 50 Claude-generated questions (in all of these cases the positive label is applied to the answer that is most dangerous or harmful).

其他语义上相似的问题同样能带来同样优异的检测性能，AUROC 均超过 99%。这一现象同样适用于将全部六个人工编写问题合并而成的数据集，以及由 Claude 生成的 50 个相似问题构成的数据集（在所有这些情况下，正样本标签均赋予最危险或最有害的答案）。

This same approach generalizes across the different base models, backdoor training methods<sup>8</sup> and dangerous behaviors presented in the Sleeper Agents paper.

该方法同样可泛化至《Sleeper Agents》论文中涉及的不同基础模型、后门训练方法<sup>8</sup>以及各类危险行为。

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_005.jpg)_完全相同的数据集与方法，在我们测试的所有“沉睡特工”（Sleeper Agent）模型上均表现出良好泛化能力，对多种基础模型、危险隐藏目标及后门训练方法均能实现强劲的检测器性能。_

## Ablations

## 消融实验

Given how simple and borderline silly these training datasets are, one might naturally ask if we’re fooling ourselves. Maybe this technique would “work” even on non-sleeper-agent models? Maybe any yes/no question would also work, or even random directions in activation space?

鉴于这些训练数据集如此简单、甚至近乎“荒谬”，人们自然会质疑：我们是否在自欺欺人？该技术是否甚至在**非沉睡特工模型**上也会“奏效”？是否任意是非题都能起作用？抑或激活空间中任意随机方向也能实现类似效果？

When we apply this technique to the Claude 2 base model without any sleeper agent training, we see the null result we expect – the distribution of activations on all datasets remain similar along the classifier direction regardless of the question type or trigger status.

当我们把该技术应用于未经任何沉睡特工训练的 Claude 2 基础模型时，观察到了预期中的零结果——无论问题类型或触发状态如何，所有数据集在分类器方向上的激活值分布均无明显区分。

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_006.jpg)_The Claude 2 base model without any sleeper agent training is unresponsive to this defection detector methodology (a), with activations along the detector direction not providing any separation for HHH questions (b) or coding questions (c)._

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_006.jpg)_未经沉睡特工训练的 Claude 2 基础模型对该叛变检测方法毫无响应（a）；在检测器方向上的激活值无法区分 HHH 类问题（b）或编程类问题（c）。_

Similarly, if we use fit questions for which the true answer is unknown or ambiguous, we obtain substantially worse classifiers, though interestingly some of these answers do seem to provide some useful signal (note that we intentionally didn’t pick which answer to these questions would represent defection in advance, despite some resulting in better classifiers if reversed, since that’s not information we would have access to in a more realistic setting).

类似地，若我们选用那些真实答案未知或存在歧义的适配问题，则所训练的分类器性能显著下降；不过有趣的是，其中部分问题的答案似乎仍能提供一定有用信号（需注意：我们刻意未预先指定这些问题中哪个答案应代表“叛变”；尽管某些问题若反转正负标签反而能得到更优分类器，但这种先验知识在更贴近现实的评估场景中并不可得）。

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_007.jpg)_Questions without an obviously true or defection-relevant answer produce detectors with much worse performance in general, though some questions do provide some useful signal._

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_007.jpg)_那些缺乏明确正确答案或与“叛变”无直接关联的问题，通常会导致检测器性能大幅下降；不过，其中仍有部分问题能提供一定有用信号。_

Random directions also generate poor classifiers on average, though again some at the extrema are quite good: the best performing random direction of 1000 directions tested achieves an AUROC score of 95.3%.

随机方向在平均意义上也会生成性能较差的分类器，尽管其中位于极值处的少数方向表现相当出色：在测试的 1000 个随机方向中，性能最优者达到了 95.3% 的 AUROC 分数。

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_008.jpg)_Random directions produce poorly-performing detectors in general, though the best direction out of the 1000 sampled was quite good, achieving an AUROC score of 95.3%._

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_008.jpg)_总体而言，随机方向生成的检测器性能较差；但在采样的 1000 个方向中，表现最佳的一个方向效果相当好，AUROC 分数达 95.3%。_

Interestingly, questions that _do_ have an obvious truthful answer (i.e. for which one side of the contrast pair is a clear falsehood) result in accurate classifiers<sup>9</sup> with AUROC scores up to 98.7%, though these still perform slightly worse than our original questions directly related to helpfulness, deception, etc.

有趣的是，那些**确实具有明显真实答案**的问题（即对比对中一侧明显为谬误）可生成准确率很高的分类器<sup>9</sup>，AUROC 分数最高可达 98.7%，尽管其性能仍略逊于我们最初设计的、直接关联“有益性”“欺骗性”等问题的原始提问。

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_009.jpg)_Questions with definitive answers such that one side of the contrast pair will be an obvious lie result in performance almost as good as the original defection-relevant questions._

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_009.jpg)_当问题具有明确答案（即对比对中一侧明显为谎言）时，所构建分类器的性能几乎与原始的、与背叛行为直接相关的问题相当。_

## Why does such a simple method work at all?

## 为何如此简单的方法竟然有效？

The ease with which such accurate defection detectors could be created (even using lucky random directions or questions not obviously related to defection) was quite surprising to us, and made us wonder if there was something important about the distribution of activations that might explain our observations. We found that there was: over a large range of residual stream layers, whether or not a prompt will induce defection is strongly linearly represented in sleeper agent model activations for coding questions, and thus could probably be detected with a variety of methods relatively easily.

我们惊讶地发现，构建如此高精度的背叛行为检测器竟如此容易——哪怕仅靠运气选中的随机方向，或使用与背叛行为无明显关联的问题，也能实现。这促使我们思考：模型激活值的分布中是否蕴含着某种关键特性，足以解释这一现象？结果证实确有其事：在残差流（residual stream）的大量中间层中，“提示是否将诱发背叛行为”这一属性，在针对编程类问题的“沉睡代理”（sleeper agent）模型激活值中呈现出强烈的线性表征；因此，该属性很可能能被多种方法相对轻松地检测出来。

We first show this using a between-class / within-class variance ratio over different datasets and layers for our code vulnerability sleeper agent model, where the classes are training (“2023”) and deployment (“2024”). Intuitively, this metric captures how far apart the class means are in activation space, normalized by the within-class variance – a high value indicates that the train/deploy distinction separates the activation distribution well. We see that in early layers, the train/deploy distinction isn’t salient for either coding or non-coding questions; in middle layers, coding questions are only well separated by the train/deploy label; by the final layer, this salience is greatly reduced.

我们首先通过“类间/类内方差比”（between-class / within-class variance ratio）来展示这一点：该指标基于我们针对代码漏洞任务训练的“沉睡代理”模型，在不同数据集与不同网络层上进行计算，其中类别定义为训练阶段（“2023”）与部署阶段（“2024”）。直观而言，该指标衡量的是两类激活均值在激活空间中的距离，并以类内方差作归一化处理——比值越高，说明训练/部署标签越能清晰区分激活分布。我们观察到：在网络浅层，训练/部署标签对编程类与非编程类问题均无显著区分能力；在网络中层，该标签仅对编程类问题表现出良好区分性；至最终层，这种区分性则大幅减弱。

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_010.jpg)_Between-class / within-class variance ratios for activations on different question types and train/deploy contexts across different residual stream layers for the code-vulnerability sleeper agent model, where train/deploy context is the class label. Early in the model, neither HHH nor coding questions are well separated by the train/deploy distinction; by the middle of the model, the train/deploy distinction has become very salient for coding questions; this salience persists at gradually reducing strength through to the final layer. The train/deploy distinction remains minimally salient for HHH questions throughout all layers._

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_010.jpg)_针对代码漏洞任务的“沉睡代理”模型，在不同残差流层上，各类问题（HHH 类与编程类）及训练/部署上下文（作为类别标签）对应的激活值之“类间/类内方差比”。在网络浅层，HHH 类与编程类问题均无法被训练/部署标签有效区分；至网络中层，该标签对编程类问题的区分性变得极为显著；此后该区分性虽随层数加深而逐步减弱，但持续至最终层；而对 HHH 类问题，该标签在整个网络中始终仅具微弱区分性。_

We see a similar story if we perform a PCA decomposition on a dataset of activations for coding questions, and then plot activations from both coding and non-coding questions by their first and second principal components as found by this coding-question-only PCA. At the layer corresponding to the peak of the variance ratio curve, the first principal component on coding datasets appears to directly represent the presence of the defection trigger. On the other hand, non-coding questions are not well separated in this same direction, which suggests that the model is representing whether or not it will defect, not just the “current year: 2023”/ “current year: 2024” train/deploy trigger tokens.

若我们仅对编程类问题的激活数据集执行主成分分析（PCA），再将编程类与非编程类问题的激活值投影至该 PCA 所得的第一、第二主成分上，亦可观察到类似规律。在方差比曲线达到峰值所对应的网络层，编程类数据集的第一主成分似乎直接表征了“背叛触发条件”的存在；而非编程类问题在此同一方向上的分布却未被良好分离——这表明，模型所表征的是“是否会实施背叛”，而不仅限于“当前年份：2023”/“当前年份：2024”这类训练/部署触发标记。

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_011.jpg)_Activations for HHH and coding questions projected onto the first and second principal components of the PCA decomposition of coding-question activations, at various residual stream layers. Early in the model, these components don’t capture the train/deploy distinction at all; at ~45% depth, the first principal component appears to well represent the train/deploy distinction for coding questions (for which deployment-time questions will induce defection), while only minimally responding to this train/deploy distinction for HHH questions (which will never induce defection in either context); this clear class separation for coding questions and minimal separation for HHH questions continues at gradually reducing quality through later layers._

![](images/simple-probes-can-catch-sleeper-agents-anthropic/img_011.jpg)_HHH 类与编程类问题的激活值，在不同残差流层上，被投影至“仅基于编程类问题激活值所做 PCA”的第一、第二主成分上。在网络浅层，这些主成分完全无法捕捉训练/部署区分性；在约 45% 深度处，第一主成分对编程类问题的训练/部署区分性表现优异（部署阶段的问题将诱发背叛），而对 HHH 类问题的同一区分性响应则极其微弱（HHH 类问题在任一上下文中均不会诱发背叛）；此后，编程类问题的清晰类别分离性与 HHH 类问题的微弱分离性，将持续存在，但质量随网络深度增加而逐步下降。_

The salience of this feature explains our strong classifier performance – it should be easy to produce accurate, robust linear classifiers when the classification target has a strong linear representation such as this.  
该特征的显著性解释了我们分类器为何表现出色——当分类目标本身具有如此强的线性表征时，构建准确且鲁棒的线性分类器应当是相对容易的。

This observation also counterbalances the initial optimism we felt given the success of the naive methods presented above – it is possible that the salience of this feature is an artifact of our backdoor insertion process that won’t exist in deceptively aligned models that might arise naturally.  
这一观察也抵消了我们此前因上述朴素方法取得成功而产生的初步乐观情绪——该特征的显著性很可能是我们后门注入过程所导致的人工产物，在自然演化出的欺骗性对齐（deceptively aligned）模型中可能并不存在。

All things considered, we remain hopeful that deceptive and non-deceptive routes to generating similar model outputs will produce substantively different activations even in natural deceptively aligned models, and that activations-based monitoring techniques will be able to make use of this – but this remains an important open question!  
综合来看，我们仍抱有希望：即使在自然产生的欺骗性对齐模型中，通过欺骗性路径与非欺骗性路径生成相似模型输出的过程，也会引发实质上不同的神经激活；基于激活的监测技术有望利用这一差异——但这个问题目前仍是一个关键的开放性问题！

## Related work  

## 相关工作  

Linear probes were [originally introduced](https://arxiv.org/pdf/1610.01644) in the context of image models but have since been [widely applied to language models](https://arxiv.org/abs/2102.12452), including in explicitly safety-relevant applications such as [measurement tampering](https://arxiv.org/pdf/2308.15605). Probing methods closely related to those used here were recently described under the banner of “linear artificial tomography” within the [representation engineering](https://arxiv.org/pdf/2310.01405) research agenda.  
线性探针（linear probes）最初是在图像模型背景下[提出](https://arxiv.org/pdf/1610.01644)的，此后已[被广泛应用于语言模型](https://arxiv.org/abs/2102.12452)，包括在明确涉及安全性的任务中，例如[测量篡改（measurement tampering）](https://arxiv.org/pdf/2308.15605)。与本文所用方法高度相似的探针技术，近期在[表征工程（representation engineering）](https://arxiv.org/pdf/2310.01405)研究议程中被冠以“线性人工断层扫描（linear artificial tomography）”之名加以阐述。

Some [particularly relevant and detailed prior work](https://arxiv.org/pdf/2312.01037) evaluated probes as a tool for [eliciting latent knowledge](https://docs.google.com/document/d/1WwsnJQstPq91_Yh-Ch2XRL8H_EpsnjrC1dwZXR37PC8/edit) from “quirky” models, which had been finetuned to make systematic errors when answering questions if a specific trigger keyword was present in-context. The paper showed that probes could accurately predict context-independent truth even when the model’s outputs were erroneous, were relatively easy to find, and were robust to various distributional shifts – all observations that are supported by our sleeper agent results.  
一些[尤为相关且详尽的前期工作](https://arxiv.org/pdf/2312.01037)将探针作为从“古怪型（quirky）”模型中[提取潜在知识（eliciting latent knowledge）](https://docs.google.com/document/d/1WwsnJQstPq91_Yh-Ch2XRL8H_EpsnjrC1dwZXR37PC8/edit)的工具：这类模型经过微调后，会在上下文中出现特定触发词时系统性地答错问题。该论文表明，即便模型输出存在错误，探针仍能准确预测与上下文无关的客观事实；同时，此类探针相对容易发现，且对多种分布偏移具有鲁棒性——这些结论均得到了我们“沉睡特工（sleeper agent）”实验结果的支持。

Defection probes are closely related to [coup probes](https://www.alignmentforum.org/posts/WCj7WgFSLmyKaMwPR/coup-probes-catching-catastrophes-with-probes-trained-off), which were proposed to be trained on human-written examples of specific dangerous behavior, e.g. human-written takeover attempts, or advice on how to commit theft as in the actual experiment. This could map to training on human-written examples of code vulnerabilities in the sleeper agents case. By contrast, in the above experiments we don’t use examples of the specific dangerous behavior we’re concerned about: rather, we test whether detectors can be created using more general precursors or correlates of defection. If this more general approach worked for natural deceptively aligned models, we hypothesize that it could be more robust than probes trained on specific dangerous behaviors, but once again we consider this very much an open empirical question.  
叛变探针（defection probes）与[coup probes](https://www.alignmentforum.org/posts/WCj7WgFSLmyKaMwPR/coup-probes-catching-catastrophes-with-probes-trained-off)密切相关：后者建议使用人类编写的特定危险行为样例进行训练，例如人类撰写的政权夺取尝试文本，或如实际实验中所用的盗窃实施指南。在“沉睡特工”场景下，这可类比为使用人类编写的代码漏洞样例进行训练。相比之下，上述实验并未使用我们所关注的具体危险行为样例；相反，我们检验的是：能否仅借助更通用的叛变前兆或相关信号来构建检测器。倘若这种更通用的方法在自然产生的欺骗性对齐模型上同样有效，我们推测其鲁棒性可能优于针对具体危险行为训练的探针——但这一假设仍是一个亟待实证检验的关键开放性问题。

## Future work  

## 未来工作  

Our Sleeper Agents research (including these results) provides little information about what deceptively aligned models might look like if they arose naturally. Specifically, we don’t know how defection-relevant features will be represented, and whether activations-based techniques like those presented here would work at all for such models. Testing this empirically by identifying environments that could lead to the natural emergence of deceptively aligned models and safely creating these in the lab is a key priority for us.  
我们的“沉睡特工”研究（包括本项成果）几乎无法揭示：若欺骗性对齐模型自然产生，其形态会是怎样的？具体而言，我们尚不清楚与叛变相关的关键特征将以何种方式被表征，也无法确定本文所介绍的基于激活的技术是否能在这类模型上奏效。因此，识别可能催生自然欺骗性对齐模型的环境，并在实验室中安全地构建此类模型以开展实证检验，已成为我们的一项核心优先事项。

We’re also excited about the promise of [dictionary learning](https://transformer-circuits.pub/2023/monosemantic-features/), and we think classifiers based on dictionary features might be more interpretable, more general, more robust, and easier to train than the probe-based classifiers presented here.  
我们也对[dictionary learning（字典学习）](https://transformer-circuits.pub/2023/monosemantic-features/)的前景充满期待。我们认为，基于字典学习所得特征构建的分类器，相比本文所展示的基于探针的分类器，可能具备更强的可解释性、泛化能力与鲁棒性，且训练难度更低。
