2024 年 9 月 2 日

![](images/restrict-repository-deployments-to-specific-teams-vercel/img_001.jpg)![](images/restrict-repository-deployments-to-specific-teams-vercel/img_002.jpg)

Enterprise customers can now restrict deployment permissions for their entire Git scope, for example their GitHub organization, to their Vercel teams.

企业客户现在可以将其整个 Git 范围（例如其 GitHub 组织）的部署权限限制为仅授权的 Vercel 团队。

Restricted deployment permissioning ensures that all repositories in the protected Git scope can only be deployed by authorized teams on Vercel.

受限部署权限机制可确保受保护 Git 范围内的所有仓库，仅能由经授权的 Vercel 团队进行部署。

This protects against intentional and accidental deployments of protected repositories.

此举可防范对受保护仓库的恶意或意外部署。

Learn more in our [documentation](https://vercel.com/docs/security/protected-git-scopes).

更多详情，请参阅我们的[文档](https://vercel.com/docs/security/protected-git-scopes)。
