## 云原生的可组合架构  

Cloud-native, composable Web stacks are becoming the go-to solution for innovative businesses. Decoupling the frontend and backend removes limitations set by monolithic platforms and frees up developers to build dynamic user experiences that convert, including:  
云原生、可组合的 Web 技术栈正日益成为创新型企业的首选方案。将前端与后端解耦，可突破单体平台的固有局限，赋能开发者构建更具转化力的动态用户体验，具体包括：

- An optimal developer workflow for the fastest release velocity  
- 面向最快发布速度的最优开发者工作流  

- Peak user experience for performance and SEO  
- 面向性能与 SEO 的极致用户体验  

- An editing environment that's decoupled from the content backend, allowing for total customization to optimize the editor experience  
- 与内容后端解耦的编辑环境，支持全面定制，以优化编辑人员的使用体验  

Take Loom: a video communication platform supporting seamless collaboration—and a joint customer of Vercel and Sanity. Their team opted to migrate from a monolithic tech stack to a composable one with Sanity, Next.js, and Vercel.  
以 Loom 为例：这是一家支持无缝协作的视频通信平台，同时也是 Vercel 与 Sanity 的共同客户。其技术团队选择从单体式技术栈迁移至由 Sanity、Next.js 和 Vercel 构建的可组合架构。

Vercel's Frontend Cloud [enables the entire Loom team](https://vercel.com/customers/loom-headless-with-nextjs) with the features they each need to ship the highest quality work. Meanwhile, their marketers are self-serving content creation and updates in Sanity, the team's CMS. “The engineering team can work on a completely separate feature or implement new designs using Next.js—all while sharing our work throughout the process,” says Tatiana Mac, a senior software engineer at Loom.  
Vercel 的前端云平台[赋能 Loom 全体团队成员](https://vercel.com/customers/loom-headless-with-nextjs)，为其提供各自所需的功能，从而交付最高质量的工作成果。与此同时，市场团队则通过 Sanity（团队所采用的 CMS）自主完成内容创作与更新。“工程团队可以完全独立地开发新功能，或借助 Next.js 实现全新设计——整个过程中，我们始终共享彼此的工作进展。”Loom 高级软件工程师 Tatiana Mac 表示。

Vercel and Sanity enable the team at Loom to scale quickly. Frontend and CMS tooling are never over-engineered for a smaller project, and engineering resources aren’t wasted when experimenting.  
Vercel 与 Sanity 共同助力 Loom 团队实现快速规模化扩展。前端与 CMS 工具不会因项目规模较小而过度设计，工程资源也不会在探索性实验中被浪费。

## Visual Editing: Vercel and Sanity’s joint solution for content collaboration

## 可视化编辑：Vercel 与 Sanity 联合推出的面向内容协作的解决方案

When you leverage monolithic architecture, modifying content directly on the server is often facilitated by a “what you see is what you get” (WYSIWYG) editor. With a composable tech stack, content management becomes decoupled from the overall system. This separation can make it difficult for authors to quickly find and replace content on their website.

当采用单体架构时，通常可通过“所见即所得”（WYSIWYG）编辑器直接在服务器端修改内容。而在可组合式技术栈中，内容管理则与整体系统解耦。这种分离可能导致内容作者难以快速定位并更新网站上的内容。

That’s why Vercel and Sanity released [Visual Editing](https://vercel.com/blog/visual-editing), which gives you a one-click path from your frontend to your content's home in Sanity Studio—so you can edit the source no matter where it may be reused.

正因如此，Vercel 与 Sanity 推出了 [可视化编辑（Visual Editing）](https://vercel.com/blog/visual-editing)，为您提供从前端页面一键直达 Sanity Studio 中内容源头的路径——无论该内容在何处被复用，您均可直接编辑其原始来源。

This means that anyone can visually edit content and experience faster iteration—from developers to marketers.

这意味着，无论是开发者还是营销人员，任何人都能通过可视化方式编辑内容，并实现更快速的迭代。
